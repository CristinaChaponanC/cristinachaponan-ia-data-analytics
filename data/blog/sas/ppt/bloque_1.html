Registro
BLOQUE 1

Proceso de registro paso a paso

1. Crear SAS Profile

  • Registro gratuito en la página de SAS
  • Cuenta general para todos los servicios SAS
  • Activar con autenticación 2FA

2. Registrarse en SAS OnDemand

  • Ingresar a welcome.oda.sas.com
  • Elegir región del servidor (US/Europe)
  • Aceptar términos de uso

3. Esperar activación

  • Proceso toma 5-10 minutos
  • SAS crea espacio de trabajo en la nube
  • Notificación por correo electrónico

4. Acceder a SAS Studio

  • Botón "Launch SAS Studio"
  • Interfaz directa en navegador
  • Conexión al servidor en la nube

Interfaz
BLOQUE 1

Navegación en SAS Studio

Panel de navegación izquierdo:

  • Files: archivos y carpetas en servidor, programas SAS (.sas)
  • Libraries: librerías WORK, SASHELP, LIBNAME personalizadas
  • Tasks & Utilities: asistentes para análisis comunes
  • Snippets: fragmentos de código reutilizables

Área de trabajo derecha:

  • Code: editor con pestañas múltiples
  • Log: mensajes NOTE/WARNING/ERROR
  • Results: tablas y gráficos generados
  • Run: botón para ejecutar código
Mostrar en vivo cada sección. Abrir SASHELP.CLASS como ejemplo.
Práctica
BLOQUE 1

Ejercicio: Verificar acceso a SAS Studio

  1. Explorar datos de ejemplo: En Libraries → SASHELP → tabla CLASS
  2. Visualizar dataset: Doble clic en CLASS para ver contenido
  3. Ejecutar primer código: En Program1, escribir y ejecutar:
PROC PRINT DATA=sashelp.class;
RUN;

Verificar:

  • Nombre de usuario visible en esquina superior
  • Resultados aparecen en pestaña Results
  • Log muestra mensajes en azul (exitoso)

Point & Click
BLOQUE 1

Tareas predefinidas vs código manual

Tasks & Utilities:

  • Asistentes para análisis comunes
  • Configuración mediante formularios
  • Generación automática de código SAS
  • Ideal para nuevos usuarios
  • Ejemplo: Summary Statistics

Proceso point-and-click:

  1. Seleccionar tarea (Statistics → Summary Statistics)
  2. Elegir dataset (sashelp.class)
  3. Configurar variables y estadísticos
  4. Revisar código generado
  5. Ejecutar y ver resultados

Ventaja: Ver el código SAS generado para aprender sintaxis

Demostrar Summary Statistics en vivo. Mostrar pestaña CODE para ver el código generado.
Fundamentos
BLOQUE 1

Librerías, Datasets y Estructura

Librerías SAS:

  • WORK: librería temporal (se borra al cerrar)
  • SASHELP: datos de ejemplo incluidos
  • Personalizadas: con LIBNAME para datos permanentes

Estructura de Dataset:

  • Observations: filas/registros
  • Variables: columnas con tipos num/char

Flujo típico:

  1. Escribir código en Editor
  2. Presionar botón Run (▶)
  3. Revisar Results para salida
  4. Verificar Log para errores/avisos
  5. Guardar scripts en Files

Solución de problemas
BLOQUE 1

Interpretando el Log y solucionando problemas

Tipos de mensajes:

  • NOTE (azul): información normal, recuentos
  • WARNING (amarillo): posibles problemas
  • ERROR (rojo): ejecución detenida

Problemas comunes:

  • Rutas de archivo incorrectas
  • Nombres de librerías/datasets
  • Permisos de archivo

Mejores prácticas:

  • Revisar Log después de cada ejecución
  • Usar obs=5 para pruebas rápidas
  • Guardar scripts periodicamente
  • Nombres claros para librerías/datasets
  • Comentar código para documentar
