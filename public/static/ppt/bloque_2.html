Conceptos fundamentales
BLOQUE 2

Data Step vs PROC Step: diferencias clave

DATA Step (Imperativo):

  • Crea o modifica datasets
  • Procesamiento registro a registro (row-wise)
  • Transformaciones personalizadas
  • Control de flujo con IF/THEN/ELSE
  • Ejemplo: crear variables, filtrar, combinar

PROC Step (Declarativo):

  • Analiza datos existentes
  • Rutinas optimizadas predefinidas
  • Menos código para tareas estándar
  • Resultados sin modificar estructura
  • Ejemplo: PRINT, SORT, MEANS, FREQ

Flujo típico: DATA para preparar → PROC para analizar

Carga de datos desde Excel/CSV en SAS OnDemand

1. Subir archivos al servidor:

  • Files (Home) → Upload
  • Seleccionar CSV/Excel desde PC
  • Verificar carga exitosa

2. Import Wizard (Point & Click):

  • File → Import Data
  • Detección automática de tipo
  • Configurar opciones y destino

3. PROC IMPORT (Código):

/* CSV */
PROC IMPORT DATAFILE="/home/usuario/datos.csv"
    OUT=work.datos
    DATAROW=2;
RUN;

/* Excel */
PROC IMPORT DATAFILE="/home/usuario/datos.xlsx"
    OUT=work.datos
    DBMS=xlsx REPLACE;
    SHEET="Hoja1";
    GETNAMES=YES;
RUN;
Demostrar ambos métodos. Siempre verificar con PROC CONTENTS después de importar.
Limpieza de datos
BLOQUE 2

Limpieza básica: valores faltantes (missing)

Detección de missing:

/* Contar missing por variable */
PROC MEANS DATA=work.datos NMISS N;
    VAR edad salario;
RUN;

/* Missing en variables carácter */
PROC FREQ DATA=work.datos;
    TABLES pais / MISSING;
RUN;

En SAS:

  • Numérico: . (punto)
  • Carácter: cadena vacía

Estrategias de tratamiento:

/* Imputar con valor fijo */
DATA work.datos_clean;
    SET work.datos;
    IF MISSING(edad) THEN edad = 30;
    IF MISSING(pais) THEN pais = 'Desconocido';
RUN;

/* Filtrar casos incompletos */
DATA work.datos_completos;
    SET work.datos;
    IF NOT MISSING(edad) AND NOT MISSING(salario);
RUN;
Enfatizar que la estrategia depende del contexto del análisis. Missing != 0.

Aplicación de formatos y uso de filtros

Formatos personalizados:

/* Crear formato personalizado */
PROC FORMAT;
    VALUE $sexoFmt
        'H' = 'Hombre'
        'M' = 'Mujer';
    VALUE rangoEdad
        0-17 = 'Menor'
        18-64 = 'Adulto'
        65-high = 'Senior';
RUN;

/* Aplicar formatos */
DATA work.datos;
    SET work.datos;
    FORMAT sexo $sexoFmt. edad rangoEdad.;
RUN;

Filtros (subsetting):

/* Filtro en DATA step */
DATA work.mayores;
    SET work.datos;
    IF edad >= 18;
RUN;

/* Filtro en PROC step */
PROC PRINT DATA=work.datos;
    WHERE sexo="M" AND salario > 50000;
RUN;

/* Filtro para casos sospechosos */
PROC PRINT DATA=work.datos;
    WHERE monto < 0;
    TITLE 'Registros con problemas';
RUN;
Diferencia entre formato permanente (DATA step) vs temporal (PROC step). WHERE vs IF.

Validación: tipos de datos y consistencia lógica

Control de tipos:

/* Verificar tipos */
PROC CONTENTS DATA=work.datos SHORT;
RUN;

/* Conversión de tipos */
DATA work.datos_fix;
    SET work.datos;
    /* Texto a número */
    edad_num = INPUT(edad_char, BEST12.);
    /* Número a texto */
    id_char = PUT(id_num, 8.);
RUN;

Validaciones lógicas:

/* Rangos válidos */
DATA work.datos_valid;
    SET work.datos;
    /* Fechas en rango lógico */
    IF NOT ('01JAN2019'd <= fecha <= TODAY())
        THEN flag_fecha = 1;
    /* Valores no negativos */
    IF monto < 0 THEN flag_monto = 1;
    /* Emails válidos (básico) */
    IF NOT INDEX(email, '@') THEN flag_email = 1;
RUN;

Principio: Detectar inconsistencias antes de que afecten el análisis

Práctica
BLOQUE 2

Ejemplos prácticos de limpieza

Normalización de texto:

/* Limpiar y estandarizar */
DATA work.datos_clean;
    SET work.datos;
    /* Normalizar país */
    pais = PROPCASE(STRIP(pais));
    /* Limpiar email */
    email = LOWCASE(STRIP(email));
    /* Crear categorías */
    IF edad < 18 THEN grupo_edad = 'Menor';
    ELSE IF edad <= 65 THEN grupo_edad = 'Adulto';
    ELSE grupo_edad = 'Senior';
RUN;

Detección de duplicados:

/* Identificar y remover duplicados */
PROC SORT DATA=work.datos 
    OUT=work.datos_nodup 
    NODUPKEY DUPOUT=work.duplicados;
    BY id_cliente;
RUN;

/* Reportar duplicados encontrados */
PROC PRINT DATA=work.duplicados;
    TITLE 'Registros duplicados detectados';
RUN;
