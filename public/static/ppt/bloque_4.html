Extract-Transform-Load
BLOQUE 4

Conceptos ETL: Extract, Transform, Load

Extract (Extracción):

  • Obtener datos de fuentes originales
  • CSV, Excel, bases de datos, APIs
  • PROC IMPORT y LIBNAME
  • Detección automática de tipos
/* Extracción de múltiples fuentes */
%let PATH = /home/&sysuserid/My Folder;

proc import datafile="&PATH/clientes.csv"
    out=work.clientes_raw dbms=csv replace;
    guessingrows=max; getnames=yes;
run;

proc import datafile="&PATH/ventas.csv"
    out=work.ventas_raw dbms=csv replace;
    guessingrows=max; getnames=yes;
run;

Transform (Transformación):

  • Limpieza y estandarización
  • Validación y reglas de negocio
  • Integración de fuentes múltiples
  • Cálculo de variables derivadas

Load (Carga):

  • Almacenamiento en destino final
  • DataMarts y data warehouses
  • Exportación en múltiples formatos
  • Reportes de proceso y auditoría

Principio: Sistemático, reproducible, auditable y escalable

ETL es la metodología estándar para mover datos desde fuentes hasta destinos analíticos, aplicando transformaciones y validaciones.

Validación Integral: Duplicados, Reglas de Negocio e Integridad

Detección de Duplicados:

/* Eliminar duplicados por clave primaria */
proc sort data=work.clientes_prep out=work.clientes_nodup 
          nodupkey dupout=work.clientes_dups; 
    by id_char; 
run;

proc sort data=work.ventas_prep out=work.ventas_nodup 
          nodupkey dupout=work.ventas_dups; 
    by id_venta; 
run;

/* Resumen de duplicados */
proc sql;
    create table work.dup_resumen as
    select 'clientes' as tabla length=16, 
           count(*) as n from work.clientes_dups 
    union all
    select 'ventas', count(*) from work.ventas_dups;
quit;

Validaciones de Reglas de Negocio:

/* Validación de ventas */
data work.ventas_chk;
    set work.ventas_nodup;
    length err_tipo $40;
    err_tipo='';
    if monto_total < 0 then err_tipo='monto_negativo';
    else if cantidad <= 0 then err_tipo='cantidad_no_positiva';
    else if missing(fecha_venta_d) then err_tipo='fecha_invalida';
    if err_tipo ne '' then output;
run;

Integridad Referencial:

/* Ventas deben referenciar clientes existentes */
proc sql;
    create table work.ventas_ref_cli as
    select v.*
    from work.ventas_nodup v
    left join work.clientes_nodup c
        on v.id_cliente_char = c.id_char
    where c.id_char is missing;
quit;
La validación es crítica en ETL. Separar datos válidos de problemáticos permite procesamiento diferenciado y trazabilidad de errores.
Construcción de DataMarts
BLOQUE 4

Integración Progresiva de Fuentes y Construcción de DataMarts

Paso 1: Enriquecer con Clientes

/* Integración 1: Ventas + Clientes */
proc sql;
    create table work.ventas_cli as
    select v.*
         , c.id_char as id_cliente
         , c.nombre_completo, c.sexo_full
         , c.pais, c.email, c.email_dominio
         , c.vendedor as vendedor_asignado
    from work.ventas_validas v
    inner join work.clientes_nodup c
        on v.id_cliente_char = c.id_char;
quit;

Paso 2: Añadir Productos

/* Integración 2: + Productos */
proc sql;
    create table work.ventas_cli_prod as
    select vc.*
         , p.categoria
         , p.precio_unitario_num as precio_unitario
    from work.ventas_cli vc
    left join work.productos_nodup p
        on strip(vc.producto) = p.producto_char;
quit;

Paso 3: Completar con Vendedores

/* Integración 3: + Vendedores */
proc sql;
    create table work.ventas_cli_prod_vend as
    select vcp.*
         , v.vendedor_char as vendedor
         , v.region as region_vendedor
         , v.fecha_ingreso_d
         , v.objetivo_mensual
    from work.ventas_cli_prod vcp
    left join work.vendedores_nodup v
        on strip(vcp.vendedor_asignado) = v.vendedor_char;
quit;

DataMart RFM con Segmentación

/* DataMart final con análisis RFM */
proc sql;
    create table work.dm_clientes as
    select c.id_char as id, c.nombre_completo
         , k.n_transacciones as F  /* Frequency */
         , k.total_monto as M      /* Monetary */
         , intck('day', k.ultima_compra, today()) as R_dias
         , case when k.total_monto >= 5000 then 'VIP'
                when k.total_monto >= 2000 then 'Premium'
                else 'Regular' end as segmento length=12
    from work.clientes_nodup c
    left join work.kpi_cliente k on c.id_char = k.id_cliente;
quit;
La integración paso a paso permite verificar la calidad en cada etapa. El DataMart RFM es fundamental para segmentación de clientes.

Aplicación de formatos y uso de filtros

Formatos personalizados:

/* Crear formato personalizado */
PROC FORMAT;
    VALUE $sexoFmt
        'H' = 'Hombre'
        'M' = 'Mujer';
    VALUE rangoEdad
        0-17 = 'Menor'
        18-64 = 'Adulto'
        65-high = 'Senior';
RUN;

/* Aplicar formatos */
DATA work.datos;
    SET work.datos;
    FORMAT sexo $sexoFmt. edad rangoEdad.;
RUN;

Filtros (subsetting):

/* Filtro en DATA step */
DATA work.mayores;
    SET work.datos;
    IF edad >= 18;
RUN;

/* Filtro en PROC step */
PROC PRINT DATA=work.datos;
    WHERE sexo="M" AND salario > 50000;
RUN;

/* Filtro para casos sospechosos */
PROC PRINT DATA=work.datos;
    WHERE monto < 0;
    TITLE 'Registros con problemas';
RUN;
Diferencia entre formato permanente (DATA step) vs temporal (PROC step). WHERE vs IF.

Parametrización y Automatización del ETL

Macrovariables para flexibilidad:

/* Parámetros configurables */
%let periodo = '31DEC2023'd;
%let umbral_monto = 1000;
%let ruta_base = /data/raw/;
%let output_lib = WORK;

/* Macro para validación genérica */
%macro validar_archivo(tabla=, campos=);
    proc contents data=&tabla out=_temp_contents noprint;
    run;
    
    proc sql noprint;
        select count(*) into :n_obs
        from &tabla;
    quit;
    
    %put INFO: Tabla &tabla tiene &n_obs observaciones;
    
    /* Verificar campos obligatorios */
    %if &campos ne %then %do;
        /* Lógica de validación */
    %end;
%mend;

ETL Automático completo:

/* Macro principal del ETL */
%macro etl_completo(fecha_proceso=);
    /* 1. Limpieza y preparación */
    %validar_archivo(tabla=RAW.ventas);
    %validar_archivo(tabla=RAW.clientes);
    
    /* 2. Transformaciones */
    proc sql;
        create table &output_lib..ventas_clean as
        select *, "&fecha_proceso"d as fecha_proceso format=date9.
        from RAW.ventas
        where fecha_venta <= "&fecha_proceso"d
          and monto_total >= &umbral_monto;
    quit;
    
    /* 3. Integración progresiva */
    %integrar_fuentes(fecha=&fecha_proceso);
    
    /* 4. Construcción DataMart */
    %construir_datamart(fecha=&fecha_proceso);
    
    /* 5. Reportes de auditoría */
    %generar_auditoria(fecha=&fecha_proceso);
%mend;

/* Ejecución */
%etl_completo(fecha_proceso=%sysfunc(today()));
La automatización permite ejecutar todo el ETL con un solo comando. Las macrovariables facilitan la configuración sin modificar código.
when total_monto >= 2000 then 'Premium' when total_monto > 0 then 'Regular' else 'Sin Compras' end as segmento length=12 , case when n_transacciones >= 10 then 'Muy Activo' when n_transacciones >= 5 then 'Activo' when n_transacciones >= 1 then 'Ocasional' else 'Inactivo' end as actividad length=12 from work.kpi_cliente as k; quit;

Uso de CALCULATED:

/* Filtrar por ticket promedio calculado */
proc sql;
    create table work.alto_ticket as
    select id, nombre_completo
         , n_transacciones
         , total_monto
         , case when n_transacciones > 0 
                then total_monto/n_transacciones 
                else . end as ticket_promedio
    from work.kpi_cliente
    having calculated ticket_promedio > 300;
quit;

Principio: CALCULATED permite referenciar columnas creadas en la misma consulta

Ventaja: Lógica condicional compleja directamente en SQL sin DATA steps adicionales

CASE WHEN es fundamental para crear categorizaciones dinámicas. CALCULATED evita subconsultas innecesarias.

DATA Step: SET/MERGE y Variables Automáticas FIRST./LAST.

MERGE - Combinación horizontal:

/* Agregar métricas por cliente */
proc sql;
    create table work.metricas_cliente as
    select id_cliente as id
         , count(*) as num_compras
         , sum(monto_total) as total_gastado
    from work.ventas_prep
    group by id_cliente;
quit;

/* Ordenar para MERGE */
proc sort data=work.clientes_prep; by id; run;
proc sort data=work.metricas_cliente; by id; run;

/* MERGE con indicadores IN= */
data work.cliente_completo;
    merge work.clientes_prep(in=tiene_datos_cliente) 
          work.metricas_cliente(in=tiene_ventas);
    by id;
    
    if tiene_datos_cliente and tiene_ventas then 
        tipo = 'Cliente Activo';
    else if tiene_datos_cliente and not tiene_ventas then 
        tipo = 'Cliente Sin Ventas';
    
    if tiene_datos_cliente; /* Solo clientes válidos */
run;

Variables FIRST./LAST. por grupos:

/* Encontrar última compra por cliente */
proc sort data=work.ventas_prep out=work.ventas_ordenadas;
    by id_cliente fecha_venta_d;
run;

data work.ultima_compra;
    set work.ventas_ordenadas;
    by id_cliente fecha_venta_d;
    
    retain fecha_ultima_compra monto_ultima_compra;
    
    if first.id_cliente then do;
        fecha_ultima_compra = .;
        monto_ultima_compra = .;
    end;
    
    /* Actualizar con cada registro */
    fecha_ultima_compra = fecha_venta_d;
    monto_ultima_compra = monto_total;
    
    /* Solo salida en último registro del cliente */
    if last.id_cliente then output;
    
    format fecha_ultima_compra date9.;
    keep id_cliente fecha_ultima_compra monto_ultima_compra;
run;

Gestión de duplicados con NODUPKEY

FIRST./LAST. son fundamentales para procesamiento por grupos. MERGE requiere ordenamiento previo pero ofrece máximo control.

Sistema de Auditoría y Monitoreo ETL

Macro de Auditoría Automatizada:

/* Sistema de auditoría completo */
%macro auditoria_etl(tabla_base=, tabla_resultado=, fecha_proceso=);
    proc sql;
        create table AUDIT.log_proceso_&fecha_proceso as
        select "&tabla_base" as tabla_origen length=50
             , "&tabla_resultado" as tabla_destino length=50
             , datetime() as timestamp_proceso format=datetime20.
             , (select count(*) from &tabla_base) as registros_origen
             , (select count(*) from &tabla_resultado) as registros_destino
             , calculated registros_destino - calculated registros_origen as diferencia
             , case when calculated diferencia = 0 then 'OK'
                    when calculated diferencia < 0 then 'PÉRDIDA'
                    else 'INCREMENTO' end as status length=12
        from (select 1 as dummy);
    quit;
    
    /* Reporte de problemas */
    %if &syserr ne 0 %then %do;
        %put ERROR: Fallo en auditoría de &tabla_base;
    %end;
%mend;

Validaciones específicas ETL:

/* Macro para detectar duplicados post-integración */
%macro check_duplicados(tabla=, key_fields=);
    proc sql;
        create table _temp_check as
        select &key_fields, count(*) as n_duplicados
        from &tabla
        group by &key_fields
        having count(*) > 1;
    quit;
    
    proc sql noprint;
        select count(*) into :n_problemas
        from _temp_check;
    quit;
    
    %if &n_problemas > 0 %then %do;
        %put WARNING: &n_problemas grupos con duplicados en &tabla;
        proc print data=_temp_check;
            title "Duplicados encontrados en &tabla";
        run;
    %end;
    %else %do;
        %put INFO: No se encontraron duplicados en &tabla;
    %end;
%mend;

/* Ejecutar auditorías */
%auditoria_etl(tabla_base=RAW.ventas, tabla_resultado=work.dm_final);
%check_duplicados(tabla=work.dm_final, key_fields=id_cliente, fecha_venta);
La auditoría automatizada es esencial para detectar problemas en producción. Los logs permiten rastrear cada transformación.

Actividad práctica: Implementar ETL completo (25 min)

Usando los 4 datasets del curso (clientes, ventas, productos, vendedores):

  1. Extract: Implementar validación de calidad con macros para cada fuente
  2. Transform: Limpiar y estandarizar datos usando las técnicas aprendidas
  3. Load: Realizar integración progresiva paso a paso (ventas+clientes, +productos, +vendedores)
  4. DataMart: Construir tabla analítica final con métricas RFM
  5. Automatización: Encapsular todo en una macro principal parametrizada
  6. Auditoría: Implementar controles de calidad y logs de proceso

Entrega: Script ETL completo, DataMart final y reporte de auditoría con métricas de calidad

Esta actividad integra todo el flujo ETL. Los participantes construirán su primer proceso automatizado de integración empresarial.
